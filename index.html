DevSecOps Foundations

What is DevSecOps

DevSecOps is the practice of integrating security into every stage of the software development lifecycle. Instead of treating security as a separate final step, it becomes a continuous responsibility shared by developers, operations, and security teams.

This approach ensures vulnerabilities are detected early, reducing risk, cost, and production incidents. Modern enterprises adopt DevSecOps to achieve both high delivery speed and strong cybersecurity posture.

DevOps vs DevSecOps

DevOps focuses on collaboration between development and operations to deliver software faster and more reliably. DevSecOps extends this model by embedding security practices such as vulnerability scanning, compliance checks, and policy enforcement into pipelines.

While DevOps improves deployment speed, DevSecOps ensures that rapid deployments do not introduce security risks or compliance violations.

Shift Left Security

Shift Left means moving security testing earlier in the development lifecycle. This ensures vulnerabilities are detected during coding rather than after deployment.

For example, detecting a hardcoded password during code review costs far less than fixing a data breach in production.

CI/CD Pipeline

Continuous Integration

Continuous Integration is the practice of frequently merging code into a shared repository where automated tests validate functionality. This reduces integration conflicts and ensures stable builds.

Example: Every Git push triggers automated testing pipelines in Jenkins or GitHub Actions.

Continuous Delivery

Continuous Delivery ensures code is always in a deployable state by automating build and test processes. Manual approval is still required before deployment to production.

This approach allows organizations to release features quickly while maintaining governance.

Continuous Deployment

Continuous Deployment automatically releases validated changes to production without human intervention. This is common in high-maturity tech organizations like Netflix.

It requires strong testing, monitoring, and rollback mechanisms to prevent service disruption.

Docker Concepts

Containers vs Virtual Machines

Virtual Machines virtualize hardware using hypervisors, resulting in higher resource consumption. Containers virtualize the OS kernel, making them lightweight and faster.

For example, a VM may take minutes to start, whereas a container starts in seconds.

Docker Security

Docker security includes image scanning, runtime protection, and least-privilege execution. Running containers as root increases risk of host compromise.

Tools like Trivy and Falco help detect vulnerabilities and suspicious runtime activities.

Kubernetes Architecture

Control Plane

The control plane manages cluster state, scheduling, and orchestration. It includes API server, scheduler, controller manager, and etcd.

Without a healthy control plane, cluster operations such as scaling or deployment will fail.

Worker Nodes

Worker nodes run containerized workloads through kubelet and container runtime. Pods are scheduled onto these nodes based on resource availability.

They form the execution layer of Kubernetes infrastructure.

DevSecOps Security

SAST

Static Application Security Testing analyzes source code without executing it. It detects vulnerabilities such as insecure coding patterns and secrets exposure.

DAST

Dynamic Application Security Testing analyzes running applications by simulating attacks. It identifies runtime vulnerabilities like injection flaws and authentication issues.

Zero Trust

Zero Trust assumes no implicit trust inside or outside the network. Every access request must be authenticated and authorized.

DevSecOps Interview Prep

1. What is DevSecOps?

Answer: DevSecOps integrates security into DevOps workflows to ensure secure software delivery. Example: Implementing SAST scanning inside Jenkins pipeline.

2. Difference between CI and CD?

Answer: CI integrates code changes, CD automates deployment readiness. Example: GitHub Actions builds code (CI) → ArgoCD deploys (CD).

3. How does Kubernetes ensure high availability?

Answer: Through ReplicaSets and self-healing scheduling.

4. What is container runtime security?

Answer: Monitoring container behavior in real-time using Falco.

5. Explain GitOps.

Answer: Using Git as the single source of truth for infrastructure.

6. What is Infrastructure as Code?

Answer: Automating infra provisioning using Terraform.

7. Difference between SAST and DAST?

Answer: SAST scans code, DAST scans running apps.

8. What is CNI?

Answer: Networking plugin for Kubernetes pods.

9. Explain Zero Trust.

Answer: Never trust network boundaries.

10. What is artifact repository?

Answer: Storage for build outputs like Docker images.

11. Explain blue-green deployment.

Answer: Two environments for zero downtime releases.

12. What is observability?

Answer: Monitoring system behavior via logs, metrics, traces.

13. What is IAM?

Answer: AWS identity and permission management.

14. Explain container escape.

Answer: When attacker breaks container isolation.

15. What is compliance?

Answer: Following standards like ISO 27001.
