01.
Foundations

What is DevSecOps?

It is the practice of integrating security at every stage of the DevOps lifecycle. It breaks the silos between Development, Security, and Operations to deliver high-quality, secure software at speed.

Shift Left Security

The strategy of moving security testing earlier in the development process. Detecting a vulnerability during the "Code" phase is significantly more cost-effective than finding it in "Production".

02.
Automated Pipeline

Continuous Integration (CI)

Automating the building and testing of code every time a developer commits to GitHub. This ensures the "Main" branch is always stable.

Continuous Deployment (CD)

Every change that passes all stages of your production pipeline is released to your customers automatically. No human intervention needed.

03.
Containers

Containers vs VMs

VMs virtualize hardware; Containers virtualize the OS. Containers are lightweight, start in seconds, and use far fewer resources than traditional Virtual Machines.

Docker Security

Implementing **Image Scanning** (Trivy), using **Non-root users**, and keeping images minimal (Distroless) to prevent container escape attacks.

04.
Orchestration

Control Plane

The Brain: kube-apiserver, etcd (store), and kube-scheduler. Manages the cluster state.

Worker Node

The Body: Kubelet and Kube-proxy. Where your Pods and containers actually run.

CNI

The Networking: Plugins like Calico and Flannel that manage pod-to-pod communication.

Sec-Ops & Prep

SAST vs DAST

SAST: Scans source code for flaws (e.g., SonarQube).
DAST: Tests the running app for vulnerabilities (e.g., OWASP ZAP).
