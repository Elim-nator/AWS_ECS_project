01.
DevSecOps
Philosophy

Continuous Security

In **QuantumShield**, security isn't a final stage; it's a constant. We use automated gates to ensure that if a Docker image contains a high-level vulnerability, the pipeline **auto-terminates** before reaching AWS.

Immutable Infrastructure

We don't "update" servers. We replace them. By using **AWS Fargate**, we deploy entirely new containers for every change, ensuring that old vulnerabilities or "configuration drift" never exist.

03.
The Execution
Roadmap

01

Containerization & Local Validation

We wrap the static HTML application into an Nginx-based Docker image. This ensures environment parity (it works on my machine AND the cloud).

CMD: docker build -t quantum-shield:v1 .
02

Secure Image Registry (ECR)

The image is pushed to Amazon ECR. ECR acts as our secure vault, automatically scanning the image for OS vulnerabilities before it is ever used.

03

The CI/CD Engine (GitHub Actions)

We configure a YAML workflow. Every time code is pushed, GitHub spins up a virtual runner, logs into AWS via OIDC/Secrets, and pushes the new image automatically.

04

Orchestration with ECS Fargate

We use Amazon ECS with the Fargate launch type. We define a 'Task Definition' that tells AWS: "Use 0.5 CPU and 1GB RAM to run this specific image."

05

Load Balancing & Public Access

An Application Load Balancer (ALB) is placed in front. It provides a single DNS URL for users and performs "Health Checks" to ensure the container is actually responding.

04.
Orchestration
Theory

K8s Architecture

Kubernetes manages clusters of containers. It consists of the Control Plane (The Brain) which manages the state, and Worker Nodes (The Muscle) where the pods live.

Self-Healing

If a container crashes in K8s, the Kubelet notices immediately and restarts it. This ensures "Zero Downtime" for applications like QuantumShield.

The War Room
Prep

Q: Why use Fargate over EC2?

"Fargate is serverless. I don't have to patch the underlying OS, manage scaling, or worry about security at the infrastructure layer—AWS handles that, allowing me to focus on the application."

Q: What is the benefit of a Load Balancer?

"It acts as a reverse proxy, distributing traffic to prevent any single container from being overwhelmed, and it provides SSL termination for secure HTTPS access."

// Cloud & Architecture

01. What is the role of an IAM Role in ECS?

"It provides the ECS Task permissions to interact with other AWS services like ECR (to pull images) or S3, without hardcoding credentials."

02. Explain the difference between Public and Private Subnets.

"Public subnets have a route to an Internet Gateway; Private subnets do not and usually require a NAT Gateway for outbound traffic."

03. What is a 'Task Definition' in AWS ECS?

"It is a blueprint for your application, defining which Docker image to use, CPU/Memory limits, and environment variables."

04. How does Auto-Scaling work in QuantumShield?

"We use CloudWatch alarms to monitor CPU usage; if it exceeds 70%, ECS automatically spins up additional Fargate tasks."

05. What is the benefit of using an Application Load Balancer (ALB)?

"It provides a single entry point, performs health checks, and supports path-based routing for microservices."

// Container Security

06. What is Image Scanning in ECR?

"It uses the Clair engine to scan images for Common Vulnerabilities and Exposures (CVEs) as soon as they are pushed."

07. Why run containers as a Non-Root user?

"To prevent 'Privilege Escalation' attacks where a compromised container could gain control over the host system."

08. What is a 'Distroless' Image?

"An image containing only your application and its dependencies, with no shell or package managers, reducing the attack surface."

09. Explain 'Container Escape'.

"A security vulnerability where a process inside a container breaks out and gains unauthorized access to the host node."

10. How do you manage secrets in a Dockerfile?

"Never hardcode them. Use BuildKit secrets or environment variables injected at runtime from a Secret Manager."

// CI/CD & Automation

11. What is a 'Self-Hosted' Runner?

"A machine you manage yourself to run GitHub Actions, providing more control over the environment and security."

12. Explain Blue-Green Deployment.

"Two identical environments; you route traffic to 'Green' only after it passes all tests, ensuring zero-downtime rollbacks."

13. What is 'GitOps'?

"Using Git as the single source of truth for infrastructure. Changes to Git automatically trigger infrastructure updates."

14. What is the purpose of a 'Build Artifact'?

"It is the output of the build process (like a .war file or Docker image) that is ready to be deployed."

15. How do you handle pipeline failures?

"Use exit codes and Slack/Email notifications to alert the team immediately, preventing bad code from progressing."

// Networking & Protocols

16. What is the difference between TCP and UDP?

"TCP is connection-oriented and reliable (3-way handshake); UDP is connectionless and fast (best for streaming)."

17. Explain the 'Least Privilege' principle.

"Giving a user or service only the minimum permissions required to perform its specific task."

18. What is an SSL/TLS Handshake?

"The process of establishing a secure, encrypted connection between a client and a server."

19. What is a 'Security Group' in AWS?

"A virtual firewall that controls inbound and outbound traffic at the instance/task level."

20. What is NAT (Network Address Translation)?

"A method of mapping multiple private IP addresses to a single public IP to allow internet access for private resources."
